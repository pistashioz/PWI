Components

Components are one of the most powerful features of Vue.js as they allow us to build reusable and maintainable code. Let's dive into the concept of components in Vue.js.

Components Overview

Components are essentially custom elements that Vue's compiler attaches behavior to. They allow us to split the user interface (UI) into independent, reusable pieces, and think about each piece in isolation.

For example, in a typical web application, we might have components for a header, sidebar, main content area, and footer. Each of these components can be written, maintained, and tested separately while working together to form a cohesive application.

Defining a Component

In Vue, a component is typically defined in a .vue file, also known as a Single-File Component (SFC). An SFC encapsulates the template, logic, and style for a component. Here is a basic structure of an SFC:

<template>
  <button @click="count++">You clicked me {{ count }} times.</button>
</template>

<script>
export default {
  data() {
    return {
      count: 0
    };
  }
}
</script>

<style scoped>
/* Component-specific styles go here */
</style>

This SFC represents a simple button component that tracks how many times it has been clicked.

Using a Component

To use a component in another component (often referred to as a "parent"), we need to:

  1. Import the component in the script part of the parent.
  1. Register it within the components option of the parent.
  1. Use it in the template as a custom tag.

Here's how we can do it:

// Import the ButtonCounter component
import ButtonCounter from './ButtonCounter.vue'

export default {
  components: {
    // Register the ButtonCounter component
    ButtonCounter
  }
}

Then in our template:

<template>
  <div>
    <h1>Here is a child component!</h1>
    <ButtonCounter />
  </div>
</template>

We can use the ButtonCounter component as many times as we like within the parent component.

Naming Conventions

It's recommended to use PascalCase for our component tags to differentiate them from native HTML elements. Although Vue.js does not enforce this convention, it is a common best practice in the Vue community.

<template>
  <div>
    <ButtonCounter />
    <ButtonCounter />
    <ButtonCounter />
  </div>
</template>

Communication between Components

Components communicate through a parent-child relationship. The parent can pass data down to its children through props, and children can send messages back up to their parents using events.

To communicate from parent to child, we use props:

<!-- ParentComponent.vue -->
<template>
  <ChildComponent message="Hello from parent!" />
</template>

To communicate from child to parent, we emit events:

// ChildComponent.vue
<script>
export default {
  methods: {
    notifyParent() {
      this.$emit('custom-event', 'Some data');
    }
  }
}
</script>

And the parent listens for those events:

<!-- ParentComponent.vue -->
<template>
  <ChildComponent @custom-event="handleCustomEvent" />
</template>

<script>
export default {
  methods: {
    handleCustomEvent(data) {
      console.log(data); // 'Some data'
    }
  }
}
</script>

This system of props down, events up, provides a clear path for data to follow, which helps maintain the one-way data flow and makes our application easier to understand and debug.

Props

Props in Vue.js are how we pass data from a parent component to a child component, making them the primary mechanism for component communication.

Basics of Props

  • Props Definition: Props are custom attributes we can register on a component. When a value is passed to a prop attribute, it becomes a property on that component instance.
  • Usage: They are used to pass data down the component tree from parent to child.

Example: Passing a Title to a Blog Post Component

Imagine we have a BlogPost component and we want to pass a title to it. We can define a prop called title to accept this piece of data.

Parent Component

import BlogPost from './components/BlogPost.vue';

export default {
  name: 'App',
  components: {
    BlogPost,
  },
};

Parent Template

<BlogPost title="My journey with Vue" />
<BlogPost title="Blogging with Vue" />
<BlogPost title="Why Vue is so fun" />

Child Component (BlogPost.vue)

export default {
  name: 'BlogPost',
  props: ['title'],
};

Child Template

<div>{{ title }}</div>

Defining Props with Types

We can define props more precisely by specifying their types. Vue.js will warn in the console if a prop receives a value of the wrong type.

props: {
  title: String,
  likes: Number,
  isPublished: Boolean,
  commentIds: Array,
  author: Object,
},

Mandatory and Default Values

Props can be made required or have a default value assigned if none is provided.

props: {
  price: {
    type: Number,
    required: true,
  },
  unit: {
    type: String,
    default: '€',
  },
},

In this example, price is required, and if not provided, Vue will issue a warning. The unit has a default value of '€'.

Validation Functions

We can provide a custom validator function as part of our prop definition. This function receives the value of the prop and should return true if the value is valid or false if it is not.

props: {
  price: {
    type: Number,
    required: true,
    validator(value) {
      return value >= 0;
    },
  },
},

This example ensures that price cannot be negative.

Reactivity with Props

Vue's reactivity system ensures that when the value of a prop changes, any part of the component template that uses that prop will automatically update.

Parent Component

export default {
  data() {
    return {
      number: 0,
    };
  },
  created() {
    setInterval(() => {
      this.number++;
    }, 1000);
  },
};

Parent Template

<DisplayNumber v-bind:number="number" />

Child Component (DisplayNumber.vue)

export default {
  props: {
    number: {
      type: Number,
      required: true,
    },
  },
};

Child Template

<div>{{ number }}</div>

Array of Objects as Props

We can also pass arrays of objects as props to display complex data structures.

Parent Component

export default {
  data() {
    return {
      castles: [{ id: 1, name: "Castle", link: "http://...", year: 1066 }],
    };
  },
};

Parent Template

<DisplayCastles v-bind:castles="castles" />

Child Component (DisplayCastles.vue)

export default {
  props: {
    castles: {
      type: Array,
      required: true,
    },
  },
};

Child Template

<table>
  <tr v-for="castle in castles" :key="castle.id">
    <td>{{ castle.id }}</td>
    <td>{{ castle.name }}</td>
    <td>{{ castle.link }}</td>
  </tr>
</table>

This setup allows DisplayCastles to render a list of castles, each represented by an object in an array, and reactivity ensures any changes to the castles array in the parent will automatically reflect in the child component.

Custom events (this.$emit)

Vue.js provides a clean and straightforward way to communicate between child and parent components using custom events with this.$emit. Here's how we can pass data from a child component to a parent component:

In the Child Component

We define a method or an inline statement that uses this.$emit to emit a custom event from the child. When calling this.$emit, the first argument is the event name, and any subsequent arguments are the data we want to pass to the parent.

Example of Child Component:

<template>
  <div>
    <input type="text" placeholder="Enter text" @input="emitToParent" />
  </div>
</template>

<script>
export default {
  methods: {
    emitToParent(event) {
      // Emit a custom event with the current value of the input
      this.$emit('input-change', event.target.value);
    }
  }
}
</script>

In the Parent Component

The parent component listens for this custom event using the v-on directive (shorthand @) in the template where the child component is used. The method in the parent that handles this event will receive the emitted data as an argument.

Example of Parent Component:

<template>
  <div>
    <!-- Listen for the custom event from the child component -->
    <child-component @input-change="handleInputChange" />
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';

export default {
  components: {
    ChildComponent
  },
  methods: {
    handleInputChange(value) {
      console.log('Value from child:', value);
      // Handle the value passed from the child component
    }
  }
}
</script>

Slots

Slots are a feature in Vue.js that allow us to compose components like we would with native HTML elements. They make it possible to insert content inside a component from the place where it is used, which can be seen as a way to pass content as props.

Normal Slots

With normal slots, we can insert content into a designated place within a component's template.

For example, consider a FancyButton component that wraps a native button element with some additional styling:

<!-- FancyButton.vue -->
<template>
  <button class="fancy-btn">
    <slot></slot> <!-- Slot outlet -->
  </button>
</template>

We can use this component in a parent and pass content to it:

<!-- Parent component -->
<FancyButton>Click me!</FancyButton>

The text "Click me!" will be inserted into the <slot></slot> of the FancyButton component, replacing the slot with the content you provide.

Named Slots

Named slots allow us to create multiple slots inside a component, each identified by a unique name.

Take a BaseLayout component as an example:

<!-- BaseLayout.vue -->
<template>
  <div class="container">
    <header>
      <slot name="header"></slot>
    </header>
    <main>
      <slot></slot> <!-- Default slot -->
    </main>
    <footer>
      <slot name="footer"></slot>
    </footer>
  </div>
</template>

In this component, there are three slots:

  • A header slot for the top of the page.
  • A default slot for the main content.
  • A footer slot for the bottom of the page.

We can use this BaseLayout component and target the slots by their names:

<!-- Parent component -->
<BaseLayout>
  <template v-slot:header>
    <h1>Here might be a page title</h1>
  </template>

  <p>A paragraph for the main content.</p>
  <p>And another one.</p>

  <template v-slot:footer>
    <p>Here's some contact info</p>
  </template>
</BaseLayout>

<!-- Shorthands -->
<BaseLayout>
  <template #header>
    <h1>Here might be a page title</h1>
  </template>

  <p>A paragraph for the main content.</p>
  <p>And another one.</p>

  <template #footer>
    <p>Here's some contact info</p>
  </template>
</BaseLayout>

In this usage:

  • The <h1> will be inserted into the header slot.
  • The two <p> tags will fill the default slot.
  • The last <p> tag will be placed into the footer slot.

AlertBox Example

A practical example of slots is creating an AlertBox component that accepts content to be displayed within the alert:

<!-- AlertBox.vue -->
<template>
  <div class="alert-box">
    <slot></slot> <!-- Content goes here -->
  </div>
</template>

We can then use the AlertBox component and pass it the content that should be displayed:

<!-- Parent component -->
<AlertBox>
  Something bad happened.
</AlertBox>

In this case, "Something bad happened." is the content passed to the AlertBox, which will be displayed where the <slot></slot> is located in the AlertBox component's template.

Reactivity of Slot Content

Just like with props, slot content is reactive. If the data passed into the slot changes in the parent component, those changes will reflect in the rendered slot content.

Slots are an incredibly powerful feature for creating complex, reusable components because they allow us to define a component's structure with placeholders for where the content will be injected. This approach maintains a clean separation between the component's structure and the content it displays, leading to more flexible and maintainable code.

Props vs Slots

Slots and props are both used to pass data to child components in Vue.js, but they serve different purposes and are used in different scenarios.

Props

  • Purpose: Props are used to pass data from a parent component to a child component. They are ideal for passing simple data down the component tree.
  • Usage: We should use props when we need to provide a child component with data or state from the parent component. Props are also reactive, so if the parent's prop value changes, the child component will automatically update to reflect the change.
  • Example: Passing a user object, a string, a number, a boolean, or any other data type to configure a child component.

Slots

  • Purpose: Slots are used to pass markup (HTML/templated content) from a parent component to a child component's template. They allow for more complex content structures that might include HTML, other components, and even scoped data.
  • Usage: Use slots when we want the parent to control a portion of the child component's content or layout. Slots are particularly useful when we need to pass complex content, like custom layouts or chunks of a user interface, that the child component will wrap or organize.
  • Example: Inserting custom buttons, links, or other HTML into a modal component.

Differences and When to Use Each

  • Complexity: Use props for simple data and slots for more complex content.
  • Control: Use props when the child component needs to react to changes in the data, and slots when the parent needs to control or format the content.
  • Reusability: If we’re creating a highly reusable component (like a generic modal or button), slots can provide the flexibility needed for various content, whereas props are better for components that always receive the same type of data.

Practical Scenario

Suppose we’re building a list component that should present items differently depending on where it's used. We would use slots to allow the parent components to define how each list item should be rendered, providing them with the scope (data related to each item). But if we also need to pass a configuration object to the list component (like a sorting function or a boolean flag to toggle a feature), we would pass that as a prop.

List Component with Props and Slots

Child Component: CustomListComponent.vue This component receives a prop for configuration and provides a slot for custom item rendering.

<template>
  <ul class="custom-list">
    <li v-for="item in items" :key="item.id">
      <!-- Use the default slot for each item, passing the item as a slot prop -->
      <slot :item="item">{{ item.defaultContent }}</slot>
    </li>
  </ul>
</template>

<script>
export default {
  name: 'CustomListComponent',
  props: {
    items: {
      type: Array,
      default: () => [],
    }
  }
}
</script>

<style>
.custom-list {
  /* Your styles here */
}
</style>
  • Default Prop Values as a Function

    In Vue.js, when we define a prop's default value to be an array or an object, we should provide a function that returns the array/object. This is to ensure that each instance of the component gets its own unique default value. If we set it directly as an empty array or object, all instances of the component could potentially share that array or object, leading to unintended side effects due to the mutable nature of arrays and objects in JavaScript.

    Here's the reason for using a function for arrays and objects:

    javascriptCopy code
props: {
  items: {
    type: Array,
    default: () => []  // Each component instance gets a unique array
  }
}

    If we used a direct array default: [], all instances would share the same array, which is usually not the behavior we want.

Parent Component Usage Example Here, we're using CustomListComponent in two different parent components, each with its own way of presenting the items.

Parent Component 1: UserListView.vue This parent component might be for a user list, with each item needing to display user details.

<template>
  <CustomListComponent :items="userList">
    <template v-slot:default="{ item }">
      <div class="user-item">
        <h2>{{ item.name }}</h2>
        <p>Email: {{ item.email }}</p>
      </div>
    </template>
  </CustomListComponent>
</template>

<script>
import CustomListComponent from './CustomListComponent.vue';

export default {
  components: {
    CustomListComponent
  },
  data() {
    return {
      userList: [
        { id: 1, name: 'Alice', email: 'alice@example.com' },
        { id: 2, name: 'Bob', email: 'bob@example.com' },
        // more users...
      ]
    };
  }
}
</script>

Parent Component 2: ProductListView.vue Another parent component might be for a product list, with each item needing to display product information.

<template>
  <CustomListComponent :items="productList">
    <template v-slot:default="{ item }">
      <div class="product-item">
        <h3>{{ item.title }}</h3>
        <p>{{ item.description }}</p>
        <p>Price: ${{ item.price }}</p>
      </div>
    </template>
  </CustomListComponent>
</template>

<script>
import CustomListComponent from './CustomListComponent.vue';

export default {
  components: {
    CustomListComponent
  },
  data() {
    return {
      productList: [
        { id: 'p1', title: 'Widget', description: 'A useful widget', price: 9.99 },
        { id: 'p2', title: 'Gadget', description: 'An essential gadget', price: 19.99 },
        // more products...
      ]
    };
  }
}
</script>

In both cases, CustomListComponent is used to render a list, but the content of each list item is determined by the parent component through the use of slots. The list items themselves are passed down as a prop (items), showcasing how both props and slots can be used together effectively.
