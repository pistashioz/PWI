Event Handling

Vue.js uses the v-on directive to listen for DOM events and execute JavaScript when they fire. For example, if we want to increase a counter each time a button is clicked, we would write:

<template>
  <div id="intro">
    <button v-on:click="counter += 1">Click me</button>
    <p>The button was clicked {{ counter }} times!</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      counter: 0
    }
  }
}
</script>

Method Event Handlers

Instead of directly writing JavaScript code within the v-on attribute, it‚Äôs often better to reference a method. This is especially true for more complex operations or when we want to reuse functionality:

<template>
  <div id="intro">
    <button v-on:click="greet">Greet</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      name: 'Vue.js'
    }
  },
  methods: {
    greet(event) {
      alert(`Hello ${this.name}`);
      if (event) {
        alert(event.target.tagName); // Shows 'BUTTON'
      }
    }
  }
}
</script>

Vue.js implicitly passes the event object to the greet method when the button is clicked. This is why we have access to event within the greet method and can use it to access properties such as event.target.tagName.

However, if we want to pass the event object explicitly or pass it along with other parameters, we use the $event variable in the template.

Inline Method Handlers

Vue also allows for inline method invocation with parameters, which can be useful for passing custom arguments directly from the template:

<template>
  <div id="intro">
    <button v-on:click="say('hello')">Say hello</button>
    <button v-on:click="say('good morning')">Say good morning</button>
  </div>
</template>

<script>
export default {
  methods: {
    say(message) {
      alert(message);
    }
  }
}
</script>

Accessing the Original Event Object

Sometimes we need access to the original DOM event. Vue.js allows us to pass it into a method using the special $event variable:

<template>
  <div id="intro">
    <button v-on:click="warn('Form cannot be submitted', $event)">Submit</button>
  </div>
</template>

<script>
export default {
  methods: {
    warn(message, event) {
      // Stop form submission
      if (event) event.preventDefault();
      alert(message);
    }
  }
}
</script>

Event Modifiers

Vue provides event modifiers as v-on directive suffixes. These modifiers tell Vue to do something before executing the event handler code, like calling event.preventDefault() or event.stopPropagation(). The available modifiers include:

.stop

The .stop modifier tells Vue to call event.stopPropagation(), which stops the event from propagating up to parent elements. It's useful when we don't want an event to trigger handlers on parent components.

<template>
  <div v-on:click="parentClick">
    Parent
    <button v-on:click.stop="childClick">Child</button>
  </div>
</template>

<script>
export default {
  methods: {
    parentClick() {
      alert("Parent clicked");
    },
    childClick() {
      alert("Child clicked");
    }
  }
}
</script>

.prevent

The .prevent modifier is used to call event.preventDefault(), preventing the default action the browser would take on that event.

<template>
  <form v-on:submit.prevent="onSubmit">
    <button type="submit">Submit</button>
  </form>
</template>

<script>
export default {
  methods: {
    onSubmit() {
      alert("Form submitted");
    }
  }
}
</script>

.capture

The .capture modifier uses event capturing instead of bubbling. In the capturing phase, events are handled by the outermost ancestor first and then propagate down to descendant elements.

<template>
  <div v-on:click.capture="captureClick">
    <button v-on:click="buttonClick">Button</button>
  </div>
</template>

<script>
export default {
  methods: {
    captureClick() {
      alert("Click captured");
    },
    buttonClick() {
      alert("Button clicked");
    }
  }
}
</script>

.self

The .self modifier ensures that the event is only triggered if event.target is the element itself, not from a child element.

<template>
  <div v-on:click.self="divClick">
    <button v-on:click="buttonClick">Button</button>
  </div>
</template>

<script>
export default {
  methods: {
    divClick() {
      alert("Div clicked directly");
    },
    buttonClick() {
      alert("Button clicked");
    }
  }
}
</script>

.once

The .once modifier makes sure that an event handler is executed only once.

<template>
  <button v-on:click.once="onceClick">Click me once</button>
</template>

<script>
export default {
  methods: {
    onceClick() {
      alert("This will only be alerted once");
    }
  }
}
</script>

.passive

The .passive modifier is used primarily to improve scrolling performance on touch and wheel events. It's a hint to the browser that you don't intend to prevent the default behavior.

<template>
  <div v-on:scroll.passive="onScroll">Scroll here</div>
</template>

<script>
export default {
  methods: {
    onScroll() {
      // Handle scrolling
    }
  }
}
</script>

The .exact Modifier

The .exact modifier in Vue.js is used to control which exact combination of system modifier keys must be pressed for an event to be triggered. It ensures that the event only fires when no other system modifiers except the specified ones are pressed.

Without the .exact modifier, the event could be triggered even if additional modifier keys are pressed. For instance, if we listen for ctrl+click, the event would fire even if the user pressed ctrl+shift+click. With .exact, we can make sure the handler only triggers with ctrl+click and nothing else.

Here are three scenarios to illustrate how .exact can be used:

  1. Only with Ctrl key:
<template>
  <!-- This will only trigger if the Ctrl key is pressed, without any other modifier keys. -->
  <button v-on:click.ctrl.exact="onCtrlClick">A</button>
</template>
  1. No Modifier Keys:
<template>
  <!-- This will only trigger if no modifier keys are pressed at all. -->
  <button v-on:click.exact="onClick">A</button>
</template>
  1. Specific Combination of Keys:
<template>
  <!-- This will trigger only if both Ctrl and Alt keys are pressed, but no others. -->
  <button v-on:click.ctrl.alt.exact="onCtrlAltClick">A</button>
</template>

Handling other events

Keyboard events other than the default ones

In Vue.js, to handle keyboard events for keys that don't have a built-in alias (like .enter, .esc, etc.), or to handle combinations of keys, we can use the keydown event along with custom logic in your methods to check which keys are pressed.

Here's how we can handle events for specific keys or combinations of keys:

  1. Listen for the keydown or keyup event.
  1. Within the event handler, check the event.keyCode or event.key to determine which key was pressed.
  1. For combinations of keys, you can check if modifier keys like event.ctrlKey, event.shiftKey, event.altKey, or event.metaKey are true at the same time as another key is pressed.

Here‚Äôs an example of handling a custom key (let's say the 'F2' key, which is often used for renaming):

<template>
  <input v-on:keydown="onKeydown" placeholder="Press F2 to rename" />
</template>

<script>
export default {
  methods: {
    onKeydown(event) {
      // Check if 'F2' was pressed
      if (event.key === 'F2' || event.keyCode === 113) {
        alert('F2 key pressed!');
      }
    }
  }
}
</script>

And here is an example of a combo (Ctrl + S, often used for saving):

<template>
  <div v-on:keydown="onKeydownCombo">Edit me and press Ctrl+S to save</div>
</template>

<script>
export default {
  methods: {
    onKeydownCombo(event) {
      // Check if 'S' was pressed along with the 'Ctrl' key
      if (event.ctrlKey && (event.key === 's' || event.keyCode === 83)) {
        event.preventDefault(); // Prevent the browser's save dialog
        alert('Ctrl+S combo pressed!');
        // Handle your custom save logic here
      }
    }
  }
}
</script>

Tab Change

Handling tab changes can be a bit more complex because it isn't just about listening to an event on an element. We might need to listen to the window or document's visibility change event if we want to detect when a user switches tabs.

<template>
  <div>
    Check the console to see when you switch tabs.
  </div>
</template>

<script>
export default {
  mounted() {
    document.addEventListener('visibilitychange', this.onTabChange);
  },
  beforeDestroy() {
    document.removeEventListener('visibilitychange', this.onTabChange);
  },
  methods: {
    onTabChange() {
      if (document.visibilityState === 'visible') {
        console.log('Tab is active');
      } else {
        console.log('Tab is inactive');
      }
    }
  }
}
</script>

Scroll Events

To handle scroll events in Vue.js, we can use the v-on directive to listen to the scroll event. Here's how we might do it:

<template>
  <div v-on:scroll="onScroll" style="overflow-y: scroll; height: 200px;">
    <!-- Content that makes the div scrollable -->
  </div>
</template>

<script>
export default {
  methods: {
    onScroll(event) {
      // Handle the scroll event
      console.log('Scrolled!', event);
    }
  }
}
</script>

For more advanced scroll interactions, such as detecting the direction of scrolling or responding to specific scroll positions, we would need to write custom logic inside our event handler. This might involve checking properties like event.deltaY (for wheel events) or the scroll position of the element (event.target.scrollTop).

Vue doesn't have built-in directives for handling mouse wheel events specifically (like scrolling up or down), so we would use the standard JavaScript events (wheel, scroll) and handle them in our methods. If we need to detect mouse wheel movements specifically, we would listen to the wheel event:

<template>
  <div v-on:wheel="onWheel" style="height: 200px; overflow-y: scroll;">
    <!-- Scrollable content -->
  </div>
</template>

<script>
export default {
  methods: {
    onWheel(event) {
      if (event.deltaY > 0) {
        console.log('Scrolled down');
      } else {
        console.log('Scrolled up');
      }
    }
  }
}
</script>

Forms Binding

In Vue.js, forms binding is a fundamental concept that allows us to create a seamless link between form inputs and our application's state. The v-model directive is at the core of this feature, enabling two-way data binding. Let's delve into what that means and how it works.

v-model

The v-model directive creates a two-way binding on form elements. When we use v-model, it:

  1. Binds an input, select, textarea, or custom component's value to a data property in our component's data object.
  1. Updates the bound data property when the user interacts with the bound form element.

This two-way data binding means that:

  • Any changes to the input field will automatically update the associated data property.
  • Any changes to the data property will automatically update the input field.

Syntax of v-model

Here's a basic example of v-model in use:

<template>
  <div>
    <input v-model="message" placeholder="Enter a message">
    <p>The message is: {{ message }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: ''
    }
  }
}
</script>

In this example:

  • We have an input element with v-model bound to the message property.
  • As the user types into the input, the message property in the component's data is updated in real time.
  • The paragraph <p> element displays the current value of the message, updating as the user types.

Checkboxes

In Vue.js, checkboxes can be bound to a data property using v-model. This binding will be either a boolean (for single checkboxes) or an array (for multiple checkboxes).

Single Checkbox

A single checkbox can be used to toggle a boolean value in our data. When the checkbox is checked, the value will be true, and when unchecked, it will be false.

<template>
  <div>
    <input type="checkbox" id="checkbox" v-model="checked">
    <label for="checkbox">{{ checked ? 'Checked' : 'Unchecked' }}</label>
  </div>
</template>

<script>
export default {
  data() {
    return {
      checked: false
    }
  }
}
</script>

Multiple Checkboxes

When dealing with multiple checkboxes, we can bind them to the same array. Each checkbox's value will be added to the array when checked and removed when unchecked.

<template>
  <div>
    <label for="ricardo">Ricardo</label>
    <input type="checkbox" id="ricardo" value="Ricardo" v-model="checkedNames">

    <label for="maria">Maria</label>
    <input type="checkbox" id="maria" value="Maria" v-model="checkedNames">

    <label for="joao">Jo√£o</label>
    <input type="checkbox" id="joao" value="Jo√£o" v-model="checkedNames">

    <span>Checked names: {{ checkedNames }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      checkedNames: []
    }
  }
}
</script>
  • To achieve a human-readable list from the array of checked names

    We can use JavaScript's array methods to construct a string in the format we want. Here's how we could modify our component to display the names in a more natural language format:

    <template>
  <div>
    <label for="ricardo">Ricardo</label>
    <input type="checkbox" id="ricardo" value="Ricardo" v-model="checkedNames">

    <label for="maria">Maria</label>
    <input type="checkbox" id="maria" value="Maria" v-model="checkedNames">

    <label for="joao">Jo√£o</label>
    <input type="checkbox" id="joao" value="Jo√£o" v-model="checkedNames">

    <span>The Checked Names are: {{ formattedCheckedNames }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      checkedNames: []
    }
  },
  computed: {
    formattedCheckedNames() {
      // Check if the array is not empty
      if (this.checkedNames.length) {
        // If only one name is checked, return it as is
        if (this.checkedNames.length === 1) {
          return this.checkedNames[0];
        }
        // Join all names with commas, but replace the last comma with 'and'
        return this.checkedNames.slice(0, -1).join(', ') + ' and ' + this.checkedNames.slice(-1);
      }
      return 'No names checked';
    }
  }
}
</script>

    In this updated example:

    • A computed property formattedCheckedNames is added, which formats the checkedNames array into a human-readable string.
    • If only one name is checked, it returns that name.
    • If multiple names are checked, it joins them with a comma and replaces the last comma with the word "and" for a proper list format.
    • If no names are checked, it returns a default message.
  • Ordering the checkedNames according to the DOM rendering

    Regarding the order of names in the array, Vue will add them to the checkedNames array in the order they are checked. So if a user checks "Jo√£o" first and then "Ricardo", the array will be ["Jo√£o", "Ricardo"]. The order is determined by user interaction, not the DOM order.

    To ensure the names in the checkedNames array are ordered according to the DOM rendering order regardless of the order in which they were checked, we would need to sort the array based on a predefined sequence that matches our DOM order. Since we're using hardcoded values for the checkboxes, we could create a reference array that defines the order.

    Here's how we can update our component to sort the checkedNames array accordingly:

    <template>
  <div>
    <label for="ricardo">Ricardo</label>
    <input type="checkbox" id="ricardo" value="Ricardo" v-model="checkedNames">

    <label for="maria">Maria</label>
    <input type="checkbox" id="maria" value="Maria" v-model="checkedNames">

    <label for="joao">Jo√£o</label>
    <input type="checkbox" id="joao" value="Jo√£o" v-model="checkedNames">

    <span>The Checked Names are: {{ orderedCheckedNames }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      checkedNames: [],
      domOrder: ['Ricardo', 'Maria', 'Jo√£o'] // Define the DOM order
    }
  },
  computed: {
    orderedCheckedNames() {
      // Sort the checked names according to the DOM order
      const sortedNames = this.checkedNames.slice().sort((a, b) => {
        return this.domOrder.indexOf(a) - this.domOrder.indexOf(b);
      });

      // Format the sorted names into a human-readable string
      if (sortedNames.length) {
        if (sortedNames.length === 1) {
          return sortedNames[0];
        }
        return sortedNames.slice(0, -1).join(', ') + ' and ' + sortedNames.slice(-1);
      }
      return 'No names checked';
    }
  }
}
</script>

    In this modified example:

    • We've added a domOrder array to data that reflects the order of the checkboxes in the DOM.
    • The orderedCheckedNames computed property now sorts the checkedNames array based on the index of each name in the domOrder array.
    • It then formats this sorted array into a human-readable string as before.

    This ensures that the checkedNames array is always sorted according to the DOM order, regardless of the order in which the user checks the boxes.

Checkboxes with Dynamic Data

For checkboxes, v-model will bind to an array when we have multiple checkboxes that represent the same array data. Here's how we can do it with dynamic data:

vueCopy code
<template>
  <div>
    <!-- Use v-for to iterate over an array of objects -->
    <div v-for="item in items" :key="item.id">
      <!-- Bind the checkbox to the 'selectedItems' array -->
      <input type="checkbox" v-model="selectedItems" :value="item.value">
      {{ item.label }}
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      selectedItems: [], // This will hold the values of the checked items
      items: [ // An array of objects, each with a 'value' and 'label'
        { id: 1, label: 'Option A', value: 'A' },
        { id: 2, label: 'Option B', value: 'B' },
        { id: 3, label: 'Option C', value: 'C' }
      ]
    }
  }
}
</script>

In this example, each checkbox is bound to the selectedItems array. When a checkbox is checked, its value is added to the array, and when it's unchecked, its value is removed.

Radio Buttons

Radio buttons allow the user to select one option from a set. When using v-model on radio buttons, the value of the selected button is bound to a data property.

<template>
  <div>
    <input type="radio" id="one" value="One" v-model="picked">
    <label for="one">One</label>

    <input type="radio" id="two" value="Two" v-model="picked">
    <label for="two">Two</label>

    <span>Picked: {{ picked }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      picked: "One"
    }
  }
}
</script>

In this radio buttons example, picked is a data property that holds the value of the selected radio button. Only one radio button can be selected at a time, and selecting a different button will update the picked value accordingly.

  • How to make Radio Buttons toggle boolean values

    We can use a radio button to toggle a boolean value, but it requires a slightly different setup compared to checkboxes. Since radio buttons are typically used for choosing between multiple options, we would need to have two radio buttons to represent true and false states. Here's an example of how we could set this up:

    <template>
  <div>
    <input type="radio" id="true" value="true" v-model="picked" :checked="picked === 'true'">
    <label for="true">True</label>

    <input type="radio" id="false" value="false" v-model="picked" :checked="picked === 'false'">
    <label for="false">False</label>

    <span>Picked: {{ picked }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      picked: 'false' // or 'true' to default to true
    }
  }
}
</script>

    In this case, picked is still a string that holds either 'true' or 'false', which are string representations of boolean values. If we want picked to be an actual boolean, we would have to convert it back and forth or use a computed property to get the boolean value:

    <script>
export default {
  data() {
    return {
      picked: 'false' // Store the selection as string
    }
  },
  computed: {
    pickedBoolean: {
      // Convert the string value to boolean
      get() {
        return this.picked === 'true';
      },
      // Convert the boolean value back to string
      set(value) {
        this.picked = value ? 'true' : 'false';
      }
    }
  }
}
</script>

    Now, pickedBoolean is a computed property that reflects the boolean value of the radio button selection. When we use this in our template or methods, we‚Äôll be working with a true boolean value, while the underlying v-model will still be using the string representation to match the radio button values.

Radio Buttons with Dynamic Data

For radio buttons, v-model will bind to a single value. Here's how we can bind dynamic values to radio buttons:

<template>
  <div>
    <!-- Use v-for to iterate over an array of objects -->
    <div v-for="item in items" :key="item.id">
      <!-- Bind the radio button to the 'selectedItem' data property -->
      <input type="radio" v-model="selectedItem" :value="item.value">
      {{ item.label }}
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      selectedItem: null, // This will hold the value of the selected item
      items: [ // An array of objects, each with a 'value' and 'label'
        { id: 1, label: 'Option A', value: 'A' },
        { id: 2, label: 'Option B', value: 'B' },
        { id: 3, label: 'Option C', value: 'C' }
      ]
    }
  }
}
</script>

In this example, selectedItem will be set to the value of the selected radio button.

Select

The <select> element is used to create a drop-down list in HTML. In Vue.js, we can bind the selected value of a <select> element to a data property using v-model. When an option is selected from the drop-down, Vue updates the bound data property, and conversely, when the bound data property changes, the selected option in the drop-down is updated.

Here's an example:

<template>
  <div>
    <select v-model="selected">
      <option disabled value="">Please select one</option>
      <option>A</option>
      <option>B</option>
      <option>C</option>
    </select>
    <span>Selected: {{ selected }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      selected: "" // The selected option is bound to this property
    }
  }
}
</script>

In this example:

  • The v-model directive binds the selected data property to the <select> element.
  • The disabled option is a placeholder and cannot be selected.
  • When the user selects an option, the selected property is updated to reflect the choice.
  • The span displays the current value of selected.

Multi-select

The <select multiple> element allows users to select multiple options from the list. We can bind it to an array using v-model. Each selected option's value is added to the array, and deselecting an option removes its value from the array.

Here's how we use it:

<template>
  <div>
    <select v-model="selected" multiple>
      <option>A</option>
      <option>B</option>
      <option>C</option>
    </select>
    <span>Selected: {{ selected }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      selected: [] // Multiple selected options are stored in this array
    }
  }
}
</script>

In this multi-select example:

  • The multiple attribute allows multiple options to be selected.
  • The v-model directive binds the selected array to the <select> element.
  • The user can select multiple options by holding down the Ctrl key (Cmd on macOS) and clicking on the options.
  • The span element displays the current array of selected options.

By using v-model with <select> and <select multiple>, we can easily implement single or multiple selections in Vue.js forms, with the selected data staying in sync with your Vue component's state.

Select with Dynamic Data

When working with dynamic data, options for a <select> element can be generated using the v-for directive. This is useful when we want to create a drop-down list based on an array of objects, where each object contains the text and value for each option.

Here's how we can implement it:

<template>
  <div>
    <select v-model="selected">
      <!-- Use v-for to dynamically generate options from the 'options' array -->
      <option v-for="option in options" v-bind:key="option.value" :value="option.value">
        {{ option.text }}
      </option>
    </select>
    <span>Selected: {{ selected }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      selected: '', // This will hold the selected value
      // Define options as an array of objects with 'text' and 'value' properties
      options: [
        { text: 'One', value: 'A' },
        { text: 'Two', value: 'B' },
        { text: 'Three', value: 'C' }
      ]
    }
  }
}
</script>

In this example:

  • The options array in the data function contains objects with text and value properties, which will be used to populate the <select> element.
  • The v-for directive on the <option> tag iterates over each item in the options array.
  • The v-bind:key binds a unique key for each option, which is necessary for Vue to keep track of each node's identity, improving performance during updates.
  • The :value is shorthand for v-bind:value, which sets the value attribute of the <option> element to the corresponding value in the options object. This is what will be assigned to the selected data property when an option is chosen.
  • The {{ option.text }} expression is used to display the text for each option.
  • The <span> displays the currently selected option's value.

By using v-for, we can easily generate a dynamic list of options for our <select> element based on our application's data, making our forms more interactive and adaptable to changing data.

Modifiers

Modifiers are special postfixes denoted by a dot, which indicate that Vue should apply special behavior to the binding. In the context of v-model, there are several modifiers, but let's focus on these three: lazy, number, and trim.

lazy

  • Functionality: By default, v-model synchronizes the input with the data after each input event (with each keystroke). The lazy modifier changes this behavior and updates the data only on the change event, which typically occurs when the user leaves the input field.
  • Example Usage:
    <input v-model.lazy="message" />

    Here, the message data property will be updated only when the input loses focus.

number

  • Functionality: This modifier automatically converts the input value to a number. Without it, even if we type a number into an input, it's treated as a string. This is particularly useful for numeric inputs.
  • Example Usage:
    <input v-model.number="age" type="number" />

    In this case, age will always be a number when updated, not a string.

trim

  • Functionality: This modifier automatically trims whitespace from the beginning and end of the input's value. It's useful for text inputs where whitespace is not meaningful or desired.
  • Example Usage:
    <input v-model.trim="username" />

    Here, any leading or trailing whitespace in username will be removed upon input.

Combining Modifiers

Modifiers can also be combined. For example, if we have an input field where we want to enter a number and don't want extra spaces:

<input v-model.number.trim="userAge" type="number" />

This setup ensures that userAge will always be a number and without any extra whitespace.

Practical Applications

In practical scenarios, these modifiers provide enhanced control over user input, ensuring that the data bound to our Vue model is as clean and as specific as our application requires. For instance, in a form where a user's age is required, using the number modifier ensures that the data type is consistent with what a backend might expect. Similarly, the trim modifier can be a time-saver in scenarios where user-generated content needs to be free of accidental whitespace, like usernames or search inputs.

Template refs

Template refs in Vue.js are a mechanism that allows us to access DOM elements directly within our Vue instances or components. They are particularly useful when we need to perform DOM operations that are not easily handled by Vue's data-driven approach.

Direct Access to DOM Elements

  • Why?: Sometimes we need to directly interact with a DOM element‚Äîfor example, to focus an input field, or to integrate with third-party libraries that require direct DOM manipulation.
  • How?: Vue provides a special ref attribute that we can place on any element within a Vue template.
    <input ref="inputRef" />

    In this snippet, inputRef is a reference name for the input element.

Accessing the Refs

  • Accessing the ref: Once the component is mounted, we can access this ref in your component's methods, computed properties, or lifecycle hooks via this.$refs, followed by the reference name.
    export default {
  mounted() {
    this.$refs.inputRef.focus();
  }
}

    Here, when the component is mounted, it automatically focuses on the input element.

  • Timing: It's important to remember that ref is not reactive and should only be accessed after the Vue component has been mounted, otherwise it will be undefined.

Refs inside v-for

  • Collection of elements: When we use ref within a v-for loop, this.$refs returns an array of elements.
    <ul>
  <li v-for="item in list" ref="items">{{ item }}</li>
</ul>

    In this example, this.$refs.items will be an array containing DOM nodes for each li element.

  • Order: The order of elements in the this.$refs.items array may not match the order of our source array due to Vue's rendering process.

Example with v-for

First, let's assume we have a simple list of names we want to render as list items:

export default {
  data() {
    return {
      // Dummy data: a list of names
      list: ['Alice', 'Bob', 'Charlie', 'Diana']
    };
  },
  mounted() {
    // When the component is mounted, log the array of refs
    console.log(this.$refs.items);
  },
  methods: {
    // A method to focus the first list item
    focusFirstItem() {
      if (this.$refs.items && this.$refs.items.length > 0) {
        this.$refs.items[0].focus(); // Focus the first element of the list
      }
    },
    // A method to apply a class to all list items
    highlightAllItems() {
      if (this.$refs.items) {
        this.$refs.items.forEach(item => {
          item.classList.add('highlight'); // Add 'highlight' class to each item
        });
      }
    }
  }
}
<template>
  <ul>
    <li v-for="item in list" ref="items" tabindex="0">{{ item }}</li>
  </ul>
  <button @click="focusFirstItem">Focus First Item</button>
  <button @click="highlightAllItems">Highlight All Items</button>
</template>

Explanation

  • Data: We have an array called list in our data function that contains four names.
  • Rendering: We use v-for to render each item in the list as an li element.
  • Refs: We assign a ref named "items" to each li element. Vue will collect these into an array.
  • Mounting: In the mounted hook, we log the array of refs to the console. This array contains the actual DOM nodes.
  • Methods:
    • focusFirstItem method checks if the items ref is present and focuses the first element of the array.
    • highlightAllItems method iterates over each item in the refs array and adds a 'highlight' class to each one.

Practical Use Case

A practical use case for template refs is when we need to integrate a Vue application with a library that manipulates the DOM directly or registers its own event listeners. Vue's reactivity system does not cover these cases, so a template ref provides the necessary bridge to the underlying DOM API.

Template refs offer a concise and convenient way to reach into the DOM when necessary, while still encouraging a data-driven approach as the primary means of DOM manipulation. It's a powerful feature that, when used judiciously, can enable advanced functionality in our Vue applications.
